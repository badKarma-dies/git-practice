# 📚 Software Engineering Lab Exercises

This repository documents my practical learning from two essential Software Engineering tools:

- 🔹 Git & GitHub – Version Control System
- 🔹 Trello – Agile Project Management

The exercises provided hands-on experience with source code management, project organization, and collaborative software development practices.

---

# 🔹 Git & GitHub

## 🎯 Objectives

The purpose of this exercise was to become familiar with Git and GitHub by performing the complete version control workflow.

The practical tasks included:

- Creating a GitHub repository
- Adding project files
- Cloning repositories locally
- Editing source code
- Reviewing code changes
- Staging and committing updates
- Publishing changes to GitHub

---

## 🚀 Learning Outcomes

By completing this exercise, I successfully learned to:

- Initialize and manage GitHub repositories.
- Clone remote repositories using Git.
- Work with local and remote repositories.
- Check repository status before committing changes.
- Inspect code modifications before staging.
- Stage individual or multiple files.
- Record project history through meaningful commits.
- Push local changes to a remote GitHub repository.
- Apply the standard Git workflow followed in professional software development.

### Git Development Cycle

```text
GitHub Repository
        │
        ▼
Clone Repository
        │
        ▼
Modify Source Code
        │
        ▼
Check Status
        │
        ▼
Review Changes
        │
        ▼
Stage Files
        │
        ▼
Create Commit
        │
        ▼
Push to GitHub
```

### Git Commands Used

| Command | Function |
|---------|----------|
| `git clone` | Download repository |
| `git status` | Display repository status |
| `git diff` | Compare modified files |
| `git add` | Stage changes |
| `git commit` | Save changes locally |
| `git push` | Upload changes to GitHub |

### Skills Developed

- Git Version Control
- Repository Management
- Code Tracking
- Local & Remote Synchronization
- Commit History Management
- Collaborative Development

# 🔹 Trello

## 🎯 Objectives

The objective of this exercise was to understand how Agile teams organize and manage software projects using Trello.

The exercise focused on:

- Creating project boards
- Managing user stories
- Organizing project workflows
- Tracking software development progress
- Applying the Kanban methodology
- Supporting Agile project management

---

## 🚀 Learning Outcomes

After completing the Trello exercise, I gained practical experience in:

- Creating and configuring Trello boards.
- Managing software requirements using user stories.
- Organizing tasks into different workflow stages.
- Tracking feature progress visually.
- Prioritizing software development tasks.
- Understanding how Agile teams collaborate.

### Trello Workflow

```text
Backlog
   │
   ▼
To Do
   │
   ▼
In Progress
   │
   ▼
Test Case Results
   │
   ▼
Done
```

### Workflow Overview

| Stage | Purpose |
|-------|---------|
| 📋 Backlog | Planned tasks waiting for development |
| 📝 To Do | Tasks selected for implementation |
| ⚙️ In Progress | Features currently under development |
| 🧪 Test Case Results | Validation and testing of completed features |
| ✅ Done | Approved and completed user stories |

### Skills Developed

- Agile Workflow Management
- User Story Management
- Sprint Planning
- Project Tracking
- Team Collaboration
- Task Prioritization
